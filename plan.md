# Codex 개발 프롬프트: 개인용 Todo macOS 앱

나만 사용할 목적의 간단한 macOS Todo 앱을 개발해줘.  
앱은 SwiftUI 기반 macOS 앱으로 만들고, 로컬 저장 방식으로 동작하면 된다.  
외부 서버, 로그인, 동기화 기능은 필요 없다.

디자인은 프로젝트 내 `DESIGN-cohere.md` 지침을 반드시 우선 적용해줘.  
UI 스타일, 컬러, 타이포그래피, 여백, 컴포넌트 톤은 해당 문서를 기준으로 맞춰줘.

---

## 1. 앱의 목적

개인 일정과 할 일을 빠르게 등록하고,  
일간 / 주간 / 월간 단위로 남은 Todo를 한눈에 확인할 수 있는 macOS Todo 앱을 만든다.

일반적인 Todo 리스트 앱보다 중요한 점은 다음이다.

- 오늘 해야 할 일을 빠르게 확인
- 이번 주에 남아 있는 일을 날짜별로 확인
- 이번 달에 남아 있는 일을 캘린더 형태로 확인
- 매주, 매월 반복되는 일을 쉽게 등록
- Todo와 별도로 간단한 메모를 남길 수 있음

---

## 2. 핵심 기능

### 2.1 Todo 기본 기능

Todo 항목은 다음 정보를 가진다.

- 제목
- 상세 설명 또는 간단 메모
- 마감일
- 완료 여부
- 중요도 또는 우선순위
- 반복 여부
- 생성일
- 수정일

기본 동작은 다음과 같다.

- Todo 추가
- Todo 수정
- Todo 삭제
- Todo 완료 처리
- 완료된 Todo 숨기기 / 보기
- 날짜별 Todo 필터링
- 남은 Todo 개수 표시

---

## 3. 화면 구조

앱은 크게 다음 화면 구조를 가진다.

### 3.1 Sidebar

왼쪽 사이드바에는 주요 메뉴를 배치한다.

- Today
- Week
- Month
- Recurring
- Notes
- Completed

각 메뉴를 선택하면 오른쪽 메인 영역이 변경된다.

---

### 3.2 Today View

오늘 날짜 기준의 Todo를 보여준다.

구성:

- 상단: 오늘 날짜, 남은 Todo 개수
- 중앙: 오늘의 Todo 리스트
- 하단 또는 우측: 빠른 Todo 추가 입력 영역

Todo 리스트 항목에는 다음 정보가 보여야 한다.

- 체크박스
- Todo 제목
- 마감 시간 또는 날짜
- 반복 여부 표시
- 우선순위 표시

Today View는 가장 자주 사용하는 화면이므로 가장 단순하고 빠르게 조작 가능해야 한다.

---

### 3.3 Week View

주간 Todo를 캘린더 방식으로 보여준다.

구성:

- 상단: 현재 주 범위 표시
- 이전 주 / 다음 주 이동 버튼
- 이번 주로 돌아가기 버튼
- 7일 컬럼 형태의 주간 캘린더

각 날짜 칸에는 다음이 표시된다.

- 날짜
- 요일
- 남은 Todo 개수
- 해당 날짜의 Todo 제목 일부
- 완료되지 않은 Todo 중심 표시

Todo가 많을 경우 모든 내용을 다 보여주지 않아도 된다.  
대신 “+3 more”처럼 추가 Todo가 있음을 표시한다.

Week View의 목적은 리스트 정리가 아니라,  
이번 주 어느 날짜에 일이 몰려 있는지 한눈에 파악하는 것이다.

---

### 3.4 Month View

월간 Todo를 캘린더 방식으로 보여준다.

구성:

- 상단: 현재 월 표시
- 이전 달 / 다음 달 이동 버튼
- 이번 달로 돌아가기 버튼
- 월간 캘린더 그리드

각 날짜 칸에는 다음이 표시된다.

- 날짜
- 남은 Todo 개수
- 주요 Todo 1~3개 정도의 짧은 제목
- 반복 Todo 표시

Month View는 “한 달 전체의 업무 밀도”를 보는 화면이다.

Todo가 많은 날짜는 시각적으로 조금 더 강조해도 된다.  
단, 디자인은 `DESIGN-cohere.md`의 톤을 유지해야 한다.

---

### 3.5 Recurring View

반복 Todo를 관리하는 화면이다.

반복 Todo는 최소 다음 타입을 지원한다.

- 매주 반복
- 매월 반복

반복 Todo 등록 시 필요한 정보:

- Todo 제목
- 설명
- 반복 주기
- 매주 반복일: 월, 화, 수, 목, 금, 토, 일 중 선택
- 매월 반복일: 매월 특정 일자 선택
- 시작일
- 종료일 optional
- 활성화 여부

반복 Todo는 해당 날짜가 되면 Today, Week, Month View에 자동으로 표시되어야 한다.

복잡한 반복 규칙은 초기 버전에서 제외한다.

제외해도 되는 기능:

- 매년 반복
- N일마다 반복
- 공휴일 제외
- 클라우드 캘린더 연동

---

### 3.6 Notes View

Todo와 별개로 간단한 메모를 작성할 수 있는 화면을 추가한다.

메모 기능은 단순하게 유지한다.

메모 데이터:

- 제목
- 본문
- 생성일
- 수정일

기본 기능:

- 메모 추가
- 메모 수정
- 메모 삭제
- 메모 리스트 보기
- 메모 상세 보기

메모는 Todo와 반드시 연결될 필요는 없다.  
초기 버전에서는 독립적인 간단 메모장으로 구현한다.

---

### 3.7 Completed View

완료된 Todo를 모아서 볼 수 있는 화면이다.

구성:

- 완료 날짜 기준 정렬
- 완료된 Todo 리스트
- 완료 취소 기능
- 삭제 기능

---

## 4. 데이터 모델 제안

다음과 같은 모델 구조로 설계해줘.

### TodoItem

- id
- title
- detail
- dueDate
- isCompleted
- priority
- recurrenceRule optional
- createdAt
- updatedAt
- completedAt optional

### RecurrenceRule

- id
- type: weekly 또는 monthly
- weekdays optional
- dayOfMonth optional
- startDate
- endDate optional
- isActive

### NoteItem

- id
- title
- body
- createdAt
- updatedAt

---

## 5. 저장 방식

초기 버전에서는 로컬 저장만 지원한다.

가능한 방식:

- SwiftData 우선 검토
- 복잡하다면 JSON 파일 기반 로컬 저장도 가능

중요한 점:

- 앱을 종료했다가 다시 열어도 Todo와 메모가 유지되어야 한다.
- 개인용 앱이므로 계정, 서버, 로그인은 만들지 않는다.

---

## 6. UI 설계 원칙

디자인은 반드시 `DESIGN-cohere.md`를 따른다.

추가적으로 다음 원칙을 적용한다.

- macOS 앱다운 사이드바 + 메인 패널 구조
- 과도한 장식보다 읽기 쉬운 정보 구조 우선
- Today View는 빠른 입력과 완료 처리 중심
- Week / Month View는 캘린더 가독성 중심
- 반복 Todo는 별도 관리 화면에서 명확하게 관리
- 메모는 Todo보다 가볍고 단순한 보조 기능으로 배치
- 전체 앱은 개인 작업 관리 도구처럼 조용하고 집중감 있게 구성

---

## 7. 에이전트 협업 방식

기획, UI-design, product-design 관점이 협의해서 결과물을 만들어줘.

각 에이전트의 역할은 다음과 같다.

### Planning Agent

- 전체 기능 범위 정리
- MVP 기준 설정
- 데이터 모델과 화면 흐름 정의
- 개인용 앱에 필요한 최소 기능과 제외 기능 구분

### UI Design Agent

- `DESIGN-cohere.md` 기준의 화면 구성
- Sidebar, Today, Week, Month, Recurring, Notes 화면 레이아웃 설계
- 캘린더 셀 안에서 Todo가 잘 보이도록 정보 밀도 조정
- macOS 앱다운 인터랙션 설계

### Product Design Agent

- 사용자가 실제로 매일 쓰는 흐름 검토
- Todo 추가, 완료, 반복 등록, 메모 작성의 사용성을 점검
- 기능이 과도하게 복잡해지지 않도록 우선순위 조정
- 개인용 생산성 앱으로서의 핵심 사용 경험 정의

세 에이전트가 협의한 뒤 다음 순서로 결과물을 만들어줘.

1. MVP 기능 정의
2. 전체 화면 구조
3. 주요 사용자 플로우
4. 데이터 모델
5. SwiftUI 컴포넌트 구조
6. 초기 폴더 구조
7. 첫 번째 구현 버전 코드

## 9. 개발 시 주의사항

- 코드는 유지보수하기 쉽게 분리해줘.
- View, Model, Store 또는 ViewModel을 명확히 나눠줘.
- 반복 Todo 생성 로직은 별도 유틸 또는 서비스로 분리해줘.
- Week / Month View의 캘린더 계산 로직도 별도 함수로 관리해줘.
- 하드코딩된 샘플 데이터는 나중에 쉽게 제거할 수 있게 해줘.
- `DESIGN-cohere.md`가 제공하는 디자인 지침과 충돌하는 스타일은 사용하지 마.

---

## 10. 기대 결과

최종적으로 다음이 가능한 앱이어야 한다.

- 앱을 열면 오늘 할 일을 바로 볼 수 있다.
- 주간 캘린더에서 날짜별 남은 일을 빠르게 볼 수 있다.
- 월간 캘린더에서 한 달의 Todo 분포를 볼 수 있다.
- 매주 또는 매월 반복되는 일을 등록할 수 있다.
- 간단한 메모를 별도로 작성하고 관리할 수 있다.
- 개인용으로 가볍고 빠르게 사용할 수 있다.

먼저 전체 설계안을 정리한 뒤, 그 설계에 따라 SwiftUI macOS 앱 코드를 생성해줘.
